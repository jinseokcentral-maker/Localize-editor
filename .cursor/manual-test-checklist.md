# 수동 테스트 체크리스트

## 📋 테스트 환경 설정

### 시작 방법

```bash
# 개발 서버 실행
pnpm dev

# 브라우저에서 접속
http://localhost:3001
```

### 기본 데이터

- 총 1005개 행 (5개 원본 + 1000개 생성)
- 언어: `en`, `ko`
- 컬럼: Row Number, Key, Context, EN, KO

## 1. 기본 그리드 기능

### 1.1 그리드 렌더링

- [ ] 그리드가 정상적으로 표시되는가?
- [ ] 헤더가 올바르게 표시되는가? (Row #, Key, Context, EN, KO)
- [ ] 데이터 행이 표시되는가?
- [ ] 행 번호 컬럼이 왼쪽에 표시되는가?

**경로**: 페이지 로드 시 자동 표시

### 1.2 스크롤

- [ ] 수직 스크롤이 부드럽게 작동하는가?
- [ ] 스크롤 시 헤더와 바디가 정렬되어 있는가?

**경로**: 마우스 휠 또는 스크롤바 사용

### 1.3 셀 편집 시작 방법

#### 1.3.1 더블클릭으로 편집 시작

- [ ] Key 셀을 더블클릭하면 편집 모드로 전환되는가?
- [ ] Context 셀을 더블클릭하면 편집 모드로 전환되는가?
- [ ] EN 셀을 더블클릭하면 편집 모드로 전환되는가?
- [ ] KO 셀을 더블클릭하면 편집 모드로 전환되는가?

**경로**:

1. 셀 더블클릭
2. 편집 모드 확인

#### 1.3.2 F2 키로 편집 시작 (키보드 전용)

- [ ] Key 셀에 포커스 후 F2를 누르면 편집 모드로 전환되는가?
- [ ] Context 셀에 포커스 후 F2를 누르면 편집 모드로 전환되는가?
- [ ] EN 셀에 포커스 후 F2를 누르면 편집 모드로 전환되는가?
- [ ] KO 셀에 포커스 후 F2를 누르면 편집 모드로 전환되는가?

**경로**:

1. Tab 또는 Arrow 키로 셀에 포커스
2. F2 키 누름
3. 편집 모드 확인

#### 1.3.3 Enter 키로 편집 시작 (키보드 전용, 언어 컬럼 제외)

- [ ] Key 셀에 포커스 후 Enter를 누르면 편집 모드로 전환되는가?
- [ ] Context 셀에 포커스 후 Enter를 누르면 편집 모드로 전환되는가?
- [ ] EN 셀에 포커스 후 Enter를 누르면 아래 행으로 이동하는가? (편집 시작 아님)
- [ ] KO 셀에 포커스 후 Enter를 누르면 아래 행으로 이동하는가? (편집 시작 아님)

**경로**:

1. Tab 또는 Arrow 키로 셀에 포커스
2. Enter 키 누름
3. 편집 모드 또는 네비게이션 확인

### 1.4 셀 편집 (Enter로 저장)

- [ ] 편집 후 Enter를 누르면 값이 저장되는가?
- [ ] 저장 후 셀 내용이 업데이트되는가?

## 2. 키보드 네비게이션

### 2.1 Tab / Shift+Tab

- [ ] Tab 키로 다음 셀로 이동하는가?
- [ ] Shift+Tab으로 이전 셀로 이동하는가?
- [ ] 행의 마지막 셀에서 Tab을 누르면 다음 행의 첫 셀로 이동하는가?
- [ ] 행의 첫 셀에서 Shift+Tab을 누르면 이전 행의 마지막 셀로 이동하는가?

**경로**:

1. 셀 클릭 또는 키보드로 포커스
2. Tab 또는 Shift+Tab 키 누름

- [ ] ↑ 키로 위 셀로 이동하는가?
- [ ] ↓ 키로 아래 셀로 이동하는가?
- [ ] ← 키로 왼쪽 셀로 이동하는가?
- [ ] → 키로 오른쪽 셀로 이동하는가?
- [ ] 첫 행에서 ↑ 키를 누르면 동작하지 않는가?
- [ ] 마지막 행에서 ↓ 키를 누르면 동작하지 않는가?

1. 셀 클릭하여 포커스
2. Arrow 키 누름

### 2.3 Enter 키 동작

- [ ] 언어 컬럼(EN, KO)에서 Enter를 누르면 아래 행으로 이동하는가?
- [ ] 언어 컬럼이 아닌 경우(Key, Context) Enter를 누르면 편집 모드로 전환되는가?

**경로**:

1. 셀에 포커스 (Tab 또는 Arrow 키)
2. Enter 키 누름
3. 동작 확인 (편집 시작 또는 네비게이션)

## 8. Goto 명령

### 8.1 행 번호로 이동

- [ ] `goto 10` 명령으로 10번째 행으로 이동하는가?
- [ ] 이동 후 해당 행이 보이는가?
- [ ] 첫 번째 편집 가능한 셀에 포커스가 설정되는가?

**경로**:

1. Cmd+K (Mac) 또는 Ctrl+K (Windows/Linux)
2. "goto 10" 입력
3. Enter로 실행

### 8.2 Top/Bottom 이동

- [ ] `goto top` 명령으로 첫 번째 행으로 이동하는가?
- [ ] `goto bottom` 명령으로 마지막 행으로 이동하는가?
- [ ] `goto first` 또는 `goto 1`도 첫 번째 행으로 이동하는가?
- [ ] `goto last`도 마지막 행으로 이동하는가?

**경로**:

1. Cmd+K (Mac) 또는 Ctrl+K (Windows/Linux)
2. "goto top" 또는 "goto bottom" 입력
3. Enter로 실행

### 8.3 Fuzzy Text Search

- [ ] `goto "keyword"` 형식으로 텍스트 검색이 가능한가?
- [ ] 따옴표 이후 텍스트가 bold/italic로 표시되는가?
- [ ] 검색 결과가 실시간으로 업데이트되는가?
- [ ] 검색 결과에서 Enter로 첫 번째 매치로 이동하는가?
- [ ] 검색 결과에서 ArrowUp/Down으로 네비게이션할 수 있는가?
- [ ] 검색 결과에서 클릭으로 이동할 수 있는가?

**경로**:

1. Cmd+K (Mac) 또는 Ctrl+K (Windows/Linux)
2. `goto "welcome"` 입력
3. 검색 결과 확인
4. Enter 또는 클릭으로 이동

### 8.4 Fuzzy Search 매칭

- [ ] Key 필드에서 매칭되는가?
- [ ] Context 필드에서 매칭되는가?
- [ ] 언어 값(EN, KO)에서 매칭되는가?
- [ ] 정확 일치가 우선순위가 높은가?
- [ ] 부분 일치도 작동하는가?

**경로**:

1. Cmd+K (Mac) 또는 Ctrl+K (Windows/Linux)
2. `goto "welcome"` 입력
3. 매칭된 필드 확인

### 8.5 Goto Next/Prev 명령

- [ ] `goto "keyword"`로 검색 후 `goto next` 명령으로 다음 매칭으로 이동하는가?
- [ ] `goto "keyword"`로 검색 후 `goto prev` 명령으로 이전 매칭으로 이동하는가?
- [ ] 마지막 매칭에서 `goto next`를 실행하면 첫 번째 매칭으로 순환하는가?
- [ ] 첫 번째 매칭에서 `goto prev`를 실행하면 마지막 매칭으로 순환하는가?
- [ ] Fuzzy find 모드에서 footer에 매칭 정보가 표시되는가? (예: "3/10 matches")
- [ ] ArrowUp/Down으로 검색 결과를 이동할 때 footer의 매칭 정보가 업데이트되는가?
- [ ] 검색 결과가 없을 때 `goto next` 또는 `goto prev`를 실행해도 오류가 발생하지 않는가?

**경로**:

1. Cmd+K (Mac) 또는 Ctrl+K (Windows/Linux)
2. `goto "button"` 입력 (여러 매칭이 있는 키워드)
3. Enter로 첫 번째 매칭으로 이동
4. 다시 Cmd+K / Ctrl+K
5. `goto next` 입력 후 Enter로 다음 매칭으로 이동
6. `goto prev` 입력 후 Enter로 이전 매칭으로 이동
7. Footer의 매칭 정보 확인 (예: "2/5 matches")

## 🔗 빠른 참조

### 키보드 단축키

- **Cmd+K / Ctrl+K**: Command Palette 열기
- **Cmd+Z / Ctrl+Z**: Undo
- **Cmd+Y / Ctrl+Y**: Redo
- **Tab**: 다음 셀로 이동
- **Shift+Tab**: 이전 셀로 이동
- **Arrow Keys**: 셀 네비게이션
- **F2**: 셀 편집 시작 (Excel 스타일)
- **Enter**:
  - 편집 저장 (편집 모드일 때)
  - 편집 시작 (Key, Context 컬럼에서 포커스만 있을 때)
  - 아래 행으로 이동 (언어 컬럼에서 포커스만 있을 때)
- **Shift+Enter**: 위 행으로 이동 (언어 컬럼)
- **Escape**: 편집 취소 / 팔레트 닫기

### 주요 명령어

- `goto <number>`: 특정 행으로 이동
- `goto top`: 첫 번째 행으로 이동
- `goto bottom`: 마지막 행으로 이동
- `goto "keyword"`: 텍스트 검색
- `goto next`: 다음 검색 결과로 이동
- `goto prev`: 이전 검색 결과로 이동
- `search <keyword>`: 키워드 검색 필터
- `filter empty`: 빈 번역 필터
- `filter changed`: 변경된 셀 필터
- `filter duplicate`: 중복 Key 필터
- `clear filter`: 필터 제거
- `undo`: 마지막 작업 취소
- `redo`: 취소한 작업 다시 적용
- `readonly`: 읽기 전용 모드 토글
- `help`: 도움말 표시
